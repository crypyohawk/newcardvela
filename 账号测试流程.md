## ⚠️ 重要：不要随便杀死所有进程！

> **绝对不要** 使用 `pkill -f copilot-api` 或 `pm2 kill` 来"重启"号池！
>
> 这样做会导致：
> 1. 孤儿进程占用端口，PM2 新实例无法绑定 → 疯狂崩溃重启（曾发生过 100+ 次重启）
> 2. copilot-api 的 wrapper 脚本使用了 `HOME` 隔离，`pkill` 会杀掉所有实例但不会清理端口
> 3. 如果 PM2 进程和孤儿进程同时跑在同一端口，只有先启动的那个有效
>
> **正确的重启方式：**
> ```bash
> # 只通过 PM2 操作，不要手动杀进程
> pm2 restart copilot-4141   # 重启单个
> pm2 restart all             # 重启所有（会重启 cardvela 等其他服务也一起）
>
> # 如果必须全部重建（紧急情况）：
> pm2 stop copilot-4141 copilot-4142 copilot-4143 copilot-4144 copilot-4145 copilot-4146
> pm2 delete copilot-4141 copilot-4142 copilot-4143 copilot-4144 copilot-4145 copilot-4146
> # 确认端口释放：
> ss -tlnp | grep 414
> # 如果还有占用，找出并杀掉：
> ps aux | grep copilot-api | grep -v grep
> kill -9 <pid>
> # 重新注册：
> pm2 start /root/copilot-homes/ecosystem.config.js
> pm2 save
> ```

## 号池架构：HOME 隔离方案

### 为什么需要 HOME 隔离

copilot-api v0.7.0 **硬编码** 数据目录为 `$HOME/.local/share/copilot-api/`（不支持 XDG_DATA_HOME）。
如果多个实例共用同一个 HOME，它们会读取同一个 `github_token` 文件，导致**所有实例只使用同一个 GitHub 账号**。
其他账号的配额完全浪费，一旦那个账号的高级模型配额耗尽，所有请求都会 402。

### 隔离方案

每个实例使用独立的 HOME 目录和 PM2 ecosystem 配置：

```
/root/copilot-homes/
├── ecosystem.config.js          # PM2 配置文件
├── 4141/
│   ├── start.sh                 # Wrapper 脚本: export HOME=... + exec copilot-api
│   └── .local/share/copilot-api/github_token   # 该账号的 token
├── 4142/
│   ├── start.sh
│   └── .local/share/copilot-api/github_token
└── ...
```

每个 `start.sh` 的内容：
```bash
#!/bin/bash
export HOME=/root/copilot-homes/<port>
exec /root/.npm/_npx/.../copilot-api start --port <port> --github-token <ghu_token>
```

### 关键 CLI 参数

- `--github-token` (别名 `-g`)：正确的 token 参数
- ~~`--token`~~：**不存在**，会被静默忽略！不要用

## 新账号接入流程

### 1. 准备独立 HOME 目录

```bash
PORT=4147  # 下一个可用端口
mkdir -p /root/copilot-homes/$PORT/.local/share/copilot-api
```

### 2. 登录 Copilot 账号并授权获取 token

```bash
npx copilot-api auth
```

按提示在浏览器中登录目标 GitHub Copilot 账号完成授权。

获取 token：
```bash
cat /root/.local/share/copilot-api/github_token
```

### 3. 写入 token 和启动脚本

```bash
TOKEN="ghu_xxxxxxx"  # 上一步获取的 token
echo -n "$TOKEN" > /root/copilot-homes/$PORT/.local/share/copilot-api/github_token

cat > /root/copilot-homes/$PORT/start.sh << EOF
#!/bin/bash
export HOME=/root/copilot-homes/$PORT
exec /root/.npm/_npx/7d9acc9b18fee6ac/node_modules/.bin/copilot-api start --port $PORT --github-token $TOKEN
EOF
chmod +x /root/copilot-homes/$PORT/start.sh
```

### 4. 更新 ecosystem.config.js

在 `/root/copilot-homes/ecosystem.config.js` 的 `apps` 数组中添加新条目：

```json
{
  "name": "copilot-4147",
  "script": "/root/copilot-homes/4147/start.sh",
  "interpreter": "bash"
}
```

### 5. 在 new-api 后台添加渠道

- 类型：OpenAI
- API 地址：`http://172.17.0.1:<port>`
- 分组：`copilot`
- 模型列表、模型映射按现有 Copilot 渠道保持一致

记下渠道 ID。

### 6. 在 CardVela 后台号池添加账号

后台号池页面添加账号，填入 githubId 和 token，绑定到上一步创建的渠道。

### 7. 启动实例

```bash
pm2 start /root/copilot-homes/ecosystem.config.js --only copilot-$PORT
pm2 save
```

### 8. 验证

```bash
ss -tlnp | grep $PORT
curl -s http://127.0.0.1:$PORT/v1/models | head -5
```

## 故障排查记录

### 2026-04-10：所有实例共享同一 GitHub 账号

**现象**：用户反馈 Claude Code 调用 API 返回 402 "You have no quota"

**根因**：copilot-api v0.7.0 硬编码 `$HOME/.local/share/copilot-api/github_token` 为数据目录。
6 个实例共用 `/root/` 作为 HOME，全部读取同一个 token 文件，导致所有实例使用 cardvela-one 账号。
cardvela-one 的 Claude 高级模型配额耗尽后，其他 5 个账号的配额完全浪费。

**修复**：HOME 隔离方案（见上文架构说明）。每个实例独立 HOME + `--github-token` 参数双重保障。

### 2026-04-10：端口冲突导致 PM2 疯狂重启

**现象**：PM2 copilot 实例重启 100+ 次，日志全是 `EADDRINUSE`

**根因**：上一次修复过程中产生了 6 个独立的 PM2 God Daemon（每个 copilot-homes/{port}/ 目录下的 .pm2），
这些孤儿 PM2 daemon 持续重启自己的 copilot-api 进程占用端口，而主 PM2 的实例无法绑定端口。

**修复**：
1. `pm2 stop` + `pm2 delete` 所有 copilot 实例
2. 杀死孤儿 PM2 God Daemon (`kill -9`)
3. 删除 `/root/copilot-homes/*/·pm2/` 目录
4. 从 ecosystem.config.js 重新注册

### 2026-04-10：claude-haiku-4-5-20251001 模型未映射

**现象**：Claude Code 请求 `claude-haiku-4-5-20251001` 返回 503 "No available channel"

**根因**：渠道只映射了 `claude-haiku-4-5` → `claude-haiku-4.5`，没有带日期后缀的变体。

**修复**：在 6 个渠道的模型列表和映射中添加 `claude-haiku-4-5-20251001` → `claude-haiku-4.5`。

## 注意事项

- 后台添加账号只是存数据库，必须启动 `copilot-api` 实例才能用
- 绑定渠道只是让 new-api 知道转发地址，端口没跑就会失败
- 每个账号对应独立端口、独立 HOME 目录和独立 new-api 渠道，不要共用
- **不要用 `pkill` 杀 copilot 进程**，只用 `pm2 restart/stop/delete` 管理
- 新增模型别名时，记得同时更新所有 6 个渠道的模型列表和映射
- HTTP 402 = 该账号高级模型配额耗尽，需要去 Copilot 后台调整额度
