## 号池启动/管理命令

以下命令均在服务器 `/opt/cardvela` 目录下执行。

### 一键重启所有实例（推荐）

从数据库读取所有 `status=active/bound` 的账号，自动启动对应端口的 `copilot-api` 实例：

```bash
cd /opt/cardvela
bash scripts/copilot-pool.sh restart
```

### 查看实例状态

```bash
bash scripts/copilot-pool.sh status
```

### 启动所有实例（不停已有的）

```bash
bash scripts/copilot-pool.sh start
```

### 停止所有实例

```bash
bash scripts/copilot-pool.sh stop
```

### 手动启动单个实例

```bash
bash scripts/copilot-pool.sh start <githubId> <token> <port>
# 例：
bash scripts/copilot-pool.sh start ferkelcheeseman@gmail.com ghu_xxxx 4144
```

### 手动停止单个实例

```bash
bash scripts/copilot-pool.sh stop <githubId>
```

## 新账号接入流程

### 1. 登录 Copilot 账号并授权获取 token

在服务器终端执行：

```bash
npx copilot-api auth
```

按提示在浏览器中登录目标 GitHub Copilot 账号完成授权。`Logged in as xxx` 只是账号标识，不是 token。

获取真正的 token：

```bash
cat /root/.local/share/copilot-api/github_token
```

### 2. 验证 token 可用

用获取到的 token 临时启动一个实例测试：

```bash
npx copilot-api start --port <port> --token <token>
```

另开一个终端测模型列表：

```bash
curl -s http://127.0.0.1:<port>/v1/models | python3 -m json.tool
```

能返回模型列表说明 token 有效。`Ctrl+C` 停掉临时实例。

### 3. 在 new-api 后台添加渠道实例

确定端口（当前下一个可用：4145），在 new-api 后台新建渠道：

- 类型：OpenAI
- API 地址：`http://172.17.0.1:<port>`
- 分组：`copilot`
- 模型列表、模型映射按现有 Copilot 渠道保持一致

记下渠道 ID。

### 4. 在 CardVela 后台号池添加账号

后台号池页面添加账号，填入 githubId 和 token。

### 5. 绑定到 new-api 渠道

在号池页面把该账号绑定到第 3 步创建的渠道。

### 6. 启动号池实例

```bash
cd /opt/cardvela
bash scripts/copilot-pool.sh restart
```

### 7. 验证

```bash
ss -lntp | grep 414
curl -s http://127.0.0.1:<port>/v1/models | python3 -m json.tool
```

## 注意事项

- 后台添加账号只是存数据库，必须启动 `copilot-api` 实例才能用
- 绑定渠道只是让 new-api 知道转发地址，端口没跑就会失败
- 每个账号对应独立端口和独立 new-api 渠道，不要共用同一个地址
