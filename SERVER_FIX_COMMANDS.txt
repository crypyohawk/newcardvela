# CardVela AI 出海工作空间 MVP 方案

更新时间：2026-04-24

目标：基于 CardVela 当前已有的虚拟卡能力，新增 AI 专用 VPN 板块，并预置教程、工作空间、AI 聚合能力入口，先跑通一个可收费、可转化、可持续迭代的 MVP。

当前阶段结论：
1. 先做虚拟卡 + VPN + 免费教程闭环。
2. VPN 先作为平台新增板块上线，但控制在付费用户可见或可购买。
3. 教程、工作空间、AI 聚合 API 先做产品预置，不在第一阶段重开发。
4. 所有设计围绕“帮助用户完成 AI 出海使用”而不是“单纯卖节点”展开。

------------------------------------------------------------
一、产品定位
------------------------------------------------------------

产品名称建议：CardVela AI 出海工作空间

一句话定位：
帮助用户一站式完成 AI 出海支付、网络、配置与学习。

目标用户：
1. 需要订阅 Claude、OpenAI、Cursor、Windsurf、GitHub Copilot 的个人用户。
2. 不熟悉国际支付、账号风控和网络环境的小白用户。
3. 想稳定使用 AI 编程工具的开发者、自由职业者、小团队。

核心卖点：
1. AI 专用虚拟卡。
2. AI 专用 VPN。
3. 免费教程和实时风控更新。
4. 后续可演进为完整的 AI 出海工作空间。

------------------------------------------------------------
二、MVP 原则
------------------------------------------------------------

1. 先卖最成熟的能力，不碰最重的风险。
2. 优先上线能直接带来付费转化的模块。
3. 页面和流程先简洁，不追求功能过多。
4. 后台先能人工运营，不要求一开始全自动。
5. 先验证用户是否愿意为“卡 + VPN + 配置便利”付费，再扩产品深度。

本阶段不做：
1. 不做 AI 模型聚合 API 中转主业务。
2. 不做复杂代理套餐体系。
3. 不做多节点智能调度。
4. 不做企业级权限系统。
5. 不做重社区和论坛功能。

------------------------------------------------------------
三、MVP 功能范围
------------------------------------------------------------

P0：第一阶段必须上线
1. 虚拟卡展示与购买入口。
2. VPN 板块。
3. 免费教程板块。
4. 用户中心里的“我的服务”聚合入口。

P1：预置但先不深做
1. AI 出海工作空间首页。
2. 一键配置包说明页。
3. 风控动态和公告页。
4. 个人推荐教程模块。

P2：占位，不在本阶段开发
1. AI 模型聚合 API 中转。
2. 自动生成多端客户端订阅。
3. 企业团队协作功能。
4. 节点多地域切换。

------------------------------------------------------------
五、P0 核心模块：VPN 板块
------------------------------------------------------------

### 1. 模块定位

定位：
AI 专用优化线路，不强调“翻墙”，强调“用于 Claude、ChatGPT、GitHub Copilot、Gmail 等 AI 与海外工具访问优化”。

模块目标：
1. 为已购卡用户提供高附加值服务。
2. 提高平台复购率和用户停留时长。
3. 形成“买卡之后继续留在平台”的第二层黏性。

### 2. 用户可见页面结构

VPN 首页建议包含以下区域：

1. 顶部权益状态区
	- 当前套餐状态
	- 到期时间
	- 是否可用
	- 在线说明
2. 一键连接区
	- Windows 下载指引
	- Android 下载指引
	- iPhone 下载指引
	- 节点链接复制按钮
	- 手动配置参数展开区
3. 推荐使用场景区
	- Claude Code
	- Cursor
	- GitHub Copilot
	- ChatGPT
	- Gmail / Google 登录
4. 使用教程区
	- Windows 配置教程
	- Android 配置教程
	- iPhone 配置教程
	- 常见错误排查
5. 服务公告区
	- 当前节点状态
	- 高峰期提醒
	- 维护通知
6. 常见问题区
	- 为什么能打开 Google 但打不开 YouTube
	- 为什么节点正常但浏览器不生效
	- 手机怎么配置
	- 是否支持多设备

### 3. VPN 权限策略

建议初期权限：
1. 免费用户不可直接使用 VPN。
2. 购买专业版套餐的用户可用。
3. 购买虚拟卡的高阶套餐可赠送 VPN。
4. 企业版用户可申请多人共享或独享线路。

建议初期运营策略：
1. 单节点先控制在 30 个稳定活跃用户规模内。
2. 对外文案可写“限量开放”。
3. 如果高峰期负载增大，再加第二台服务器。
4. 先人工发放与续期，后台保留自动化升级空间。

### 4. VPN 页面功能清单

用户侧功能：
1. 查看 VPN 是否已开通。
2. 查看套餐有效期。
3. 复制节点链接。
4. 查看各端教程。
5. 查看节点状态公告。
6. 查看常见问题。
7. 查看最近更新记录。

运营侧功能：
1. 开通用户 VPN 权限。
2. 设置开始时间和到期时间。
3. 设置套餐类型。
4. 发布节点变更公告。
5. 临时关闭或维护提示。
6. 查看授权用户列表。
7. 预留多节点字段。

### 6. VPN 用户流程

用户购买后流程建议：
1. 用户购买卡或专业版套餐。
2. 系统判断是否包含 VPN 权益。
3. 用户中心出现“我的 VPN”入口卡片，不在原用户页内联展开内容。
4. 用户点击后跳转到独立 VPN 页面，查看节点链接、配置教程和状态说明。
5. 用户按系统或手机类型完成配置。
6. 用户使用 Claude / Gmail / GitHub 等场景。
7. 到期前 3 天推送续费提醒。

### 7. VPN 页面文案方向

推荐标题：
1. AI 专用 VPN
2. 更适合 Claude、Cursor、Gmail 的稳定访问线路

推荐说明文案：
1. 本服务面向 AI 工具与海外工作流优化。
2. 适合 Claude Code、GitHub Copilot、ChatGPT、Google 登录等场景。
3. 已内置 Windows、Android、iPhone 配置教程。
4. 线路名额限量开放，保障稳定性。

### 8. VPN MVP 验收标准

1. 登录用户能看到 VPN 板块入口。
2. 无权限用户看到购买或升级引导。
3. 有权限用户能看到节点链接和配置教程。
4. 节点公告可后台修改并前台显示。
5. 管理员可开通和关闭单个用户的 VPN 权限。
6. 页面可支持后续新增第二个节点而不重做结构。

九、P2 预置模块：AI 模型聚合 API 中转
------------------------------------------------------------

当前策略：
1. 导航可预留“即将上线”入口。
2. 不作为 MVP 主功能开发。
3. 不对外承诺上线时间。
4. 等现金流和来源稳定后再评估。

原因：
1. 风控高。
2. 资金压力大。
3. 服务稳定性要求高。
4. 容易分散当前最该做的业务精力。

十、定价建议
------------------------------------------------------------

建议采用三层结构：

1. 基础用户
	- 可购买虚拟卡
	- 可免费查看教程
	- 不含 VPN

2. 专业版
	- 虚拟卡
	- VPN 权益
	- 配置教程优先入口
	- 风控动态提醒

3. 企业版
	- 更高额度
	- 专属支持
	- 后续可增加专属节点或企业服务

当前建议：
1. VPN 可以做独立增值项。
2. 也可以作为专业版打包权益。
3. 第一阶段以“买卡送 VPN”或“升级专业版赠 VPN”更容易促成转化。

4. 这样可以避免用户页文件继续膨胀，降低后续编辑和维护成本。

前端建议优先新增页面：
1. /vpn

用户中心建议新增卡片：
1. 我的 VPN：点击跳转 /vpn
2. 我的教程：点击跳转 /tutorials
3. 风控公告：点击跳转 /workspace 或公告页

第一阶段页面实现优先级：
1. 先做 VPN 独立页面。
2. 再做教程列表页。
3. 再做工作空间聚合页。
4. AI 聚合 API 页面先做占位。

------------------------------------------------------------
十三、风险与应对
------------------------------------------------------------

风险一：VPN 高峰期卡顿
应对：
1. 初期限量开放。
2. 默认只面向付费用户。
3. 负载接近阈值时再加第二节点。

风险二：教程流量高但付费转化低
应对：
1. 每篇教程加入自然转化入口。
2. 把“支付 + 网络 + 教程”组合成完整解决方案。
3. 把 VPN 权益和专业版捆绑。

风险三：产品过早做重
应对：
1. 先只上 VPN。
2. 教程先做内容页。
3. 工作空间先做聚合，不做复杂自动化。

------------------------------------------------------------
十四、当前执行建议
------------------------------------------------------------

当前最优先事项：
1. 先设计并上线 /vpn 独立页面。
2. 用户体系里预留 VPN 权限字段。
3. 后台增加 VPN 权限开关和公告能力。
4. 用户页只保留 VPN 跳转入口，避免继续膨胀。
5. 教程中心先做列表结构和占位分类。
6. 工作空间页面先做信息聚合原型。

一句话执行顺序：
先让用户能在平台上点击入口进入独立 VPN 页面并完成配置使用，再逐步补教程和工作空间。

十六、VPN 独立页面开发方案
------------------------------------------------------------

页面定位：
1. /vpn 是用户查看和使用 VPN 的唯一业务页面。
2. 用户中心不再承载 VPN 详情，只负责跳转。
3. 这样可以把复杂逻辑从大文件中剥离出来，降低维护成本。

建议路由：
1. 前台用户页入口：/dashboard 或用户中心中的“我的 VPN”卡片
2. 独立页面：/vpn
3. 可选详情页：后续如有多节点，再扩为 /vpn/nodes 或 /vpn/history

页面状态拆分：
1. 未登录：提示登录
2. 已登录无权限：展示购买或升级引导
3. 已登录有权限：展示节点、教程、公告、FAQ
4. 节点维护中：展示维护公告和恢复时间说明

页面模块拆分建议：
1. Hero 区
   - 标题
   - 当前权益状态
   - 到期时间
   - 购买/续费按钮
2. 节点信息区
   - 节点名称
   - 节点链接复制按钮
   - 手动配置参数折叠面板
3. 客户端下载区
   - Windows
   - Android
   - iPhone
4. 配置教程区
   - Windows 教程
   - Android 教程
   - iPhone 教程
5. 公告区
   - 维护通知
   - 高峰提醒
   - 最近更新
6. FAQ 区
   - YouTube 打不开
   - 浏览器不走代理
   - 手机配置方式

组件拆分建议：
1. VPNStatusCard
2. VPNNodeCard
3. VPNConfigAccordion
4. VPNClientDownloadCard
5. VPNNoticeList
6. VPNFaqList
7. VPNPurchaseGuide

推荐数据获取方式：
1. 页面首屏请求一个聚合接口，避免前端多次串行请求。
2. 公告和 FAQ 可拆独立接口，但首版更建议聚合返回。
3. 用户中心只请求一个 very light summary 接口，用于显示是否开通和按钮跳转。

------------------------------------------------------------
十三、风险与应对
------------------------------------------------------------

风险一：VPN 高峰期卡顿
应对：
1. 初期限量开放。
2. 默认只面向付费用户。
3. 负载接近阈值时再加第二节点。

风险二：教程流量高但付费转化低
应对：
1. 每篇教程加入自然转化入口。
2. 把“支付 + 网络 + 教程”组合成完整解决方案。
3. 把 VPN 权益和专业版捆绑。

风险三：产品过早做重
应对：
1. 先只上 VPN。
2. 教程先做内容页。
3. 工作空间先做聚合，不做复杂自动化。

------------------------------------------------------------
十四、当前执行建议
------------------------------------------------------------

当前最优先事项：
1. 先设计并上线 VPN 板块页面。
2. 用户体系里预留 VPN 权限字段。
3. 后台增加 VPN 权限开关和公告能力。
4. 教程中心先做列表结构和占位分类。
5. 工作空间页面先做信息聚合原型。

一句话执行顺序：
先让用户能在平台上看到、购买、进入、配置并使用 VPN，再逐步补教程和工作空间。
