===== 当前有效步骤 (v3 - 彻底修复) =====

问题根因:
1. 之前平台自己生成 key 传给 new-api，但 new-api 在 POST 创建 token 时会改写 key（加前缀/重新生成）。
2. 之前的修复虽然做了回查，但仍先传一个自生成 key，存在 new-api 改写后回查拿到脱敏值的风险。
3. 本次彻底修复：不传 key 给 new-api，让 new-api 自己生成 key，直接保存创建响应里的真实 key。
4. 旧的 key 不会自动变好，必须删除后重新创建。


步骤1 - 服务器拉取最新代码:
cd ~/cardvela
git pull


步骤2 - 重新安装、构建、重启:
rm -rf .next
npm install
npm run build
pm2 restart cardvela


步骤3 - 删除旧测试 key:
说明: 用户后台里之前失败过的 key 先删除，再重新创建全新的 key。


步骤4 - 用户端重新创建一个全新的 key:
说明: 这一步创建出来的新 key 才是修复后的 key。
