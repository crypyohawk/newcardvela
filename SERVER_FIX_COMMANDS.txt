===== 用量同步扣费修复 (2026-03-28) =====

问题根因:
  设计文档要求每5分钟从 new-api 拉取用量日志并扣费，但定时同步任务从未实现。
  getNewApiLogs() 函数已写好却从未被调用，webhook 等着被推送但无推送者。
  导致用户有使用记录但后台无扣费。

修改清单:
  1. src/lib/usageSync.ts - 新建：提取扣费核心逻辑为共享模块
  2. app/api/cron/sync-usage/route.ts - 新建：cron端点，主动从new-api拉取日志并扣费
  3. app/api/webhook/new-api/route.ts - 简化为调用共享模块
  4. src/lib/newapi.ts - 修复getNewApiLogs()响应格式兼容（支持两种new-api版本）

安全修复:
  - 幂等性检查移入事务内，防止并发双重扣费
  - 每条日志独立try-catch，单条失败不中断整批
  - 移除Prisma查询的 as any 强制断言

部署步骤:

  # 1. 拉取并构建
  cd ~/cardvela
  git pull origin main
  npm run build
  pm2 restart all

  # 2. 验证服务启动
  pm2 logs cardvela --lines 20

  # 3. 手动测试一次同步（用 NEW_API_WEBHOOK_SECRET 的值替换）
  curl "http://localhost:3000/api/cron/sync-usage?secret=你的SECRET"

  # 4. 配置 crontab 每5分钟自动同步
  crontab -e
  # 添加这行（把 你的SECRET 替换为 NEW_API_WEBHOOK_SECRET 或 CRON_SECRET 的值）:
  */5 * * * * curl -s "http://localhost:3000/api/cron/sync-usage?secret=你的SECRET" >> /home/ubuntu/cardvela/sync.log 2>&1

注意事项:
  - CRON_SECRET 默认复用 NEW_API_WEBHOOK_SECRET 环境变量，也可单独设置 CRON_SECRET
  - 首次运行最多回溯24小时日志，之后每次从上次同步时间戳开始
  - 同步时间戳存储在 SystemConfig 表 key=usage_sync_last_timestamp
  - sync.log 会记录每次同步结果，可用于排查

===== 查看/设置 SECRET 以及配置 crontab 详细步骤 =====

  # 1. 查看服务器上是否已有 NEW_API_WEBHOOK_SECRET
  grep NEW_API_WEBHOOK_SECRET ~/cardvela/.env

  # 2. 如果上面输出为空，说明从未配置，需要生成一个并写入：
  echo "NEW_API_WEBHOOK_SECRET=$(openssl rand -hex 32)" >> ~/cardvela/.env

  # 3. 再次查看确认已写入，复制等号后面的值
  grep NEW_API_WEBHOOK_SECRET ~/cardvela/.env

  # 4. 写入后必须重启让环境变量生效
  cd ~/cardvela && pm2 restart all

  # 5. 手动测试一次同步（把下面的 你的SECRET 替换为第3步看到的值）
  curl "http://localhost:3000/api/cron/sync-usage?secret=你的SECRET"
  # 正常返回示例: {"success":true,"fetched":15,"synced":12,"skipped":2,"duplicated":1,"totalCostDeducted":0.0834}
  # 如果返回 {"error":"未授权"} 说明 secret 不对，重新检查第3步
  # 如果返回 {"message":"无新日志","synced":0} 说明连接正常只是暂无日志

  # 6. 配置 crontab 每5分钟自动同步
  crontab -e
  # 编辑器打开后，在文件最末尾添加下面这一行（替换 你的SECRET）:
  */5 * * * * curl -s "http://localhost:3000/api/cron/sync-usage?secret=你的SECRET" >> /home/ubuntu/cardvela/sync.log 2>&1
  # 保存退出: nano编辑器按 Ctrl+X -> Y -> Enter

  # 7. 验证 crontab 已注册
  crontab -l

  # 8. 等5分钟后查看同步日志
  tail -20 ~/cardvela/sync.log
